# 修改计划

## 目标

按最小风险、可验证、可回滚的顺序，逐步解决 `review-3-27.md` 中仍未关闭的问题，并在每一阶段完成后更新本计划。

## 总原则

1. 先修关键路径：状态迁移、生命周期、risk monitor。
2. 先加兼容层，再迁移调用点，最后删旧接口。
3. 不做大重写；每次只解决一类问题并补测试。
4. 所有修改都必须通过 `go test ./...`、`go vet ./...`、`go build ./...`。

## 执行顺序

### Phase 1：关键路径安全

- [x] 1. 引入 typed position patch / command，逐步替代 `UpdatePosition(... map[string]any)`。
  - 目标文件：`internal/store/*`、`internal/position/position_service.go`、`internal/reconcile/*`、`internal/position/risk_plan_service.go`
  - 验收：关键状态迁移不再直接拼动态 map；保留兼容层。

  - 目标文件：`internal/market/metrics_service.go`、`internal/runtime/runtime_builder_compression.go`、`internal/runtime/*`
  - 验收：builder 不再直接启动 goroutine；Start/Stop ownership 明确。

- [x] 3. risk monitor 改成 cache-first + bounded fallback。
  - 目标文件：`internal/position/risk_monitor.go`、相关测试文件
  - 验收：风险 tick 不再把远端 IO 当作常规主路径；fallback 有明确超时和 freshness 策略。

### Phase 2：规范收口

- [x] 4. 收口 symbol normalization 到统一 helper，清理本次涉及路径里的 ad-hoc normalize。
  - 目标文件：`internal/pkg/symbol/symbol.go`、`internal/bootstrap/symbols.go`、`internal/market/metrics_service.go`、`internal/position/plan_cache.go`、`internal/reconcile/*`
  - 验收：本次触达模块不再新增 `strings.ToUpper(strings.TrimSpace(...))` 风格的 symbol 归一化。

- [x] 5. registry 改成显式构建，移除 `init + panic` 业务注册副作用。
  - 目标文件：`internal/risk/initexit/*`
  - 验收：默认策略集合通过显式 builder 构建；测试可构造局部 registry。

### Phase 3：结构减负

- [x] 6. 收敛 bootstrap/runtime builder 中的重复依赖组装。
  - 目标文件：`internal/bootstrap/*`、`internal/runtime/*`
  - 验收：runtime/service 依赖组装路径更少、更集中。

- [x] 7. 继续压缩 decisionfmt 剩余大 helper，并补测试。
  - 目标文件：`internal/decision/decisionfmt/*`
  - 验收：热点 helper 更小，golden/table-driven tests 覆盖关键输出契约。
